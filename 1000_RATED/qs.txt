CODING #1 - WEB CRAWLER
High Frequency | Software Engineer
================================================================================
 
BASIC PROBLEM
-------------
You are given:
- A starting URL startUrl
- An interface HtmlParser that can fetch all URLs from a given web page
 
Implement a web crawler that returns all URLs reachable from startUrl that share
the same hostname as startUrl. The order of URLs in the result does not matter.
 
Interface:
  interface HtmlParser {
      public List<String> getUrls(String url);
  }
 
Your function: List<String> crawl(String startUrl, HtmlParser htmlParser)
 
REQUIREMENTS
- Start from startUrl
- Use HtmlParser.getUrls(url) to obtain all links from a page
- Avoid duplicates — never crawl the same URL twice
- Restrict scope — only follow URLs whose hostname matches that of startUrl
- Assume all URLs use the http protocol and do not include a port
 
NOTES FOR CANDIDATE
- URL Fragments: Consider how to handle URL fragments (e.g., http://example.com/page#section1).
  Should these be treated as the same URL or different?
- URL Normalization: Assume URL normalization is NOT required for the basic problem.
 
FOLLOW-UP: MULTITHREADED/CONCURRENT IMPLEMENTATION (Important!!)
-----------------------------------------------------------------
After implementing the basic single-threaded version, implement a multithreaded
or concurrent version to improve performance.
 
Requirements:
- Parallelize the crawling — multiple URLs should be fetched concurrently
- Thread safety — ensure no race conditions when accessing shared data structures
- Avoid duplicates — even with multiple threads, each URL should only be crawled once
- Maintain hostname restriction
 
NOTE: Use a thread pool to control the total number of threads.
- Do NOT create unbounded threads (e.g., one thread per URL)
- Common approach: Use a fixed-size thread pool (e.g., 10-20 threads) with a task queue
 
SOLUTION: JavaScript Multithreaded Implementation
-------------------------------------------------
 
class ThreadPool {
    constructor(maxConcurrency) {
        this.maxConcurrency = maxConcurrency;
        this.queue = [];
        this.runningTasks = 0;
        this.waitForFinishPromise = null;
        this.waitForFinishResolve = null;
    }
 
    execute(task) {
        return new Promise((resolve, reject) => {
            this.queue.push({ task, resolve, reject });
            this._processQueue();
        });
    }
 
    _processQueue() {
        if (this.runningTasks >= this.maxConcurrency) return;
        if (this.queue.length === 0) {
            if (this.runningTasks === 0) {
                this.waitForFinishResolve && this.waitForFinishResolve();
                this.waitForFinishPromise = null;
                this.waitForFinishResolve = null;
            }
            return;
        }
        const { task, resolve, reject } = this.queue.shift();
        this.runningTasks++;
        Promise.resolve()
            .then(task)
            .then(resolve, reject)
            .finally(() => {
                this.runningTasks--;
                this._processQueue();
            });
    }
 
    waitForFinish() {
        if (this.runningTasks === 0 && this.queue.length === 0) {
            return Promise.resolve();
        }
        if (!this.waitForFinishPromise) {
            this.waitForFinishPromise = new Promise((resolve) => {
                this.waitForFinishResolve = resolve;
            });
        }
        return this.waitForFinishPromise;
    }
}
 
function getProcessedUrlObject(url) {
    const newUrl = new URL(url);
    newUrl.hash = "";  // Remove hash fragments
    return newUrl;
}
 
async function crawl(startUrl, htmlParser, maxConcurrency = 10) {
    const processedStartUrlObject = getProcessedUrlObject(startUrl);
    const startHostName = processedStartUrlObject.hostname;
 
    const result = [processedStartUrlObject.toString()];
    const visited = new Set([processedStartUrlObject.toString()]);
    const pool = new ThreadPool(maxConcurrency);
 
    const crawlUrl = async (url) => {
        try {
            const nextUrls = await htmlParser.getUrls(url);
            for (const nextUrl of nextUrls) {
                const nextUrlProcessedObject = getProcessedUrlObject(nextUrl);
                const nextUrlProcessedString = nextUrlProcessedObject.toString();
                if (nextUrlProcessedObject.hostname === startHostName
                    && !visited.has(nextUrlProcessedString)) {
                    visited.add(nextUrlProcessedString);
                    result.push(nextUrlProcessedString);
                    pool.execute(() => crawlUrl(nextUrl)).catch((err) => {
                        console.error(`Error crawling ${nextUrl}:`, err);
                    });
                }
            }
        } catch (error) {
            console.error(`Error fetching URLs from ${url}:`, error);
        }
    };
 
    await pool.execute(() => crawlUrl(startUrl));
    await pool.waitForFinish();
    return result;
}

CODING #2 - DEDUPLICATE FILES
High Frequency | Software Engineer, Machine Learning Engineer
================================================================================
 
BASIC PROBLEM
-------------
Given a root folder/directory, find all duplicate files within it. Return groups
of file paths where each group contains files with identical content.
 
Example directory structure:
  root/
  ├── a/
  │   ├── f1.mp4
  │   └── f2.mp4
  └── b/
      ├── f3.mp4
      └── tmp/
          └── f4.mp4
 
Expected output:
  [
      ["a/f1.mp4", "a/f2.mp4"],        # Group 1: identical content
      ["b/f3.mp4", "b/tmp/f4.mp4"]     # Group 2: identical content
  ]
 
REQUIREMENTS
- Traverse all files in the directory tree starting from the root folder
- Group files by content
- Return only duplicates (files that appear alone not included)
- Handle nested directories recursively
 
INTERFACE
  def find_duplicate_files(root_path: str) -> List[List[str]]:
 
NOTES FOR CANDIDATE
- Create test files in the interview environment (e.g., Google Colab)
- Consider edge cases: empty files, symbolic links, file permissions
- Time complexity depends on total size of all files
 
OPTIMIZED APPROACH: Three-Stage Filtering
-----------------------------------------
Stage 1 - File Size (Metadata):
  - Group files by size first
  - Files with different sizes cannot be duplicates
  - Very fast (just metadata lookup, no I/O)
 
Stage 2 - Partial Hash (First Chunk):
  - For files with the same size, read and hash only the first chunk (e.g., 1024 bytes)
  - Files with different partial hashes cannot be duplicates
 
Stage 3 - Full Hash:
  - Only when size and partial hash match, compute the full content hash
 
Implementation structure:
  def find_duplicate_files_optimized(root_path: str) -> List[List[str]]:
      size_groups = group_by_file_size(root_path)
      partial_hash_groups = {}
      for files in size_groups.values():
          if len(files) > 1:
              group_by_partial_hash(files, partial_hash_groups)
      full_hash_groups = {}
      for files in partial_hash_groups.values():
          if len(files) > 1:
              group_by_full_hash(files, full_hash_groups)
      return [files for files in full_hash_groups.values() if len(files) > 1]
 
ADDITIONAL DISCUSSION TOPICS (verbal)
--------------------------------------
1. I/O Bound vs CPU Bound
   - Usually I/O bound (reading files from disk is slower than computing hashes)
   - With modern SSDs, may become more CPU bound
   - If I/O bound: Use async I/O, read files in parallel
   - If CPU bound: Use faster hash algorithms (xxHash), parallelize hash computation
 
2. Hash Function Selection
   - MD5: Fast but cryptographically broken; acceptable collision rate for dedup
   - SHA-256: Slower but very low collision probability
   - xxHash/CityHash: Extremely fast non-cryptographic hashes
   - For most cases: SHA-256 provides excellent collision resistance
 
3. Distributed System Implementation
   - MapReduce Approach: Map (compute hashes), Shuffle (group same hashes), Reduce (output duplicates)
   - Partitioning Strategy: Partition by file size first
 
4. Continuous Monitoring System
   - File System Watcher: Use OS-level file system events (inotify, FSEvents)
   - Database Schema: hash_to_files, file_to_hash tables
   - Notification System: Queue-based (RabbitMQ, Kafka)
 
5. Chunk Size Selection
   - Factors: I/O throughput, early differentiation, memory usage, disk sector size
   - Typical values: 1KB-4KB for quick check, 64KB-1MB for SSDs

CODING #3 - CONVERTING STACK SAMPLES TO TRACE EVENTS
High Frequency | Software Engineer, Machine Learning Engineer
================================================================================
 
BASIC PROBLEM
-------------
A sampling profiler periodically records the full call stack at a timestamp.
Given a list of samples sorted by timestamp, convert them into trace events.
 
Struct definitions:
  struct Sample {
      double ts;
      std::vector<std::string> stack;  // outermost → innermost
  };
 
  struct Event {
      std::string kind;   // "start" or "end"
      double ts;
      std::string name;
  };
 
Example:
  Input: t=1.0: ["main"], t=2.5: ["main", "func1"], t=3.1: ["main"]
  Output:
    start 1.0 main
    start 2.5 func1
    end   3.1 func1
 
REQUIREMENTS
- Between consecutive samples, determine what changed
- End functions that disappeared (deepest-first order)
- Start functions that newly appeared (outer-to-inner order)
- Do NOT automatically close remaining functions after the last sample
- Handle recursion correctly — same function name at different depths = separate frames
 
INTERFACE
  List<Event> convertSamplesToEvents(List<Sample> samples)
 
NOTE: Find the longest common prefix to determine which functions ended/started
 
BASIC SOLUTION
--------------
  def convert_samples_to_events(samples):
      events = []
      prev_stack = []
 
      for sample in samples:
          ts, stack = sample.ts, sample.stack
          common_len = 0
          for i in range(min(len(prev_stack), len(stack))):
              if prev_stack[i] == stack[i]:
                  common_len += 1
              else:
                  break
 
          # End functions that disappeared (deepest first)
          for i in range(len(prev_stack) - 1, common_len - 1, -1):
              events.append(Event("end", ts, prev_stack[i]))
 
          # Start functions that newly appeared (outer to inner)
          for i in range(common_len, len(stack)):
              events.append(Event("start", ts, stack[i]))
 
          prev_stack = stack
      return events
 
Walkthrough:
  Step | Previous Stack      | Current Stack       | Common | Ends          | Starts
  1    | []                  | ["main"]            | 0      | —             | start 1.0 main
  2    | ["main"]            | ["main","func1"]    | 1      | —             | start 2.5 func1
  3    | ["main","func1"]    | ["main"]            | 1      | end 3.1 func1 | —
 
Recursion: Handled correctly because we compare by position, not name.
 
Complexity: Time O(S×D), Space O(D) where S=samples, D=max stack depth
 
FOLLOW-UP 1: DEBOUNCING BY N CONSECUTIVE SAMPLES
-------------------------------------------------
Only emit start/end events for functions that appear in N consecutive samples.
 
Definition of "Consecutive":
  - Same frame (same depth and ancestors) appears in N samples in a row
  - If a function ends and later restarts, its streak RESETS
 
Example (N=2):
  t=1.0: ["main"]
  t=2.0: ["main"]
  t=3.0: ["main", "foo"]
  t=4.0: ["main", "foo"]
  Output:
    start 2.0 main   # confirmed after 2 samples
    start 4.0 foo    # confirmed after 2 samples
 
Critical Edge Case:
  Sample(1, ['a', 'b']) then Sample(2, ['c', 'b', 'a'])
  → a and b do NOT count as consecutive (common prefix = 0, since 'a' != 'c')
 
INTERFACE
  List<Event> convertSamplesToDebouncedEvents(List<Sample> samples, int N)
 
SOLUTION
--------
  def convert_samples_to_debounced_events(samples, N):
      events = []
      prev_stack = []
      streak_counts = []
      confirmed = []
 
      for sample in samples:
          ts, stack = sample.ts, sample.stack
          common_len = 0
          for i in range(min(len(prev_stack), len(stack))):
              if prev_stack[i] == stack[i]:
                  common_len += 1
              else:
                  break
 
          # End confirmed functions that disappeared (deepest first)
          for i in range(len(prev_stack) - 1, common_len - 1, -1):
              if confirmed[i]:
                  events.append(Event("end", ts, prev_stack[i]))
 
          new_streaks = []
          new_confirmed = []
 
          # Common prefix: increment streaks
          for i in range(common_len):
              new_streak = streak_counts[i] + 1
              new_streaks.append(new_streak)
              if new_streak >= N and not confirmed[i]:
                  events.append(Event("start", ts, stack[i]))
                  new_confirmed.append(True)
              else:
                  new_confirmed.append(confirmed[i])
 
          # New functions: start with streak of 1
          for i in range(common_len, len(stack)):
              new_streaks.append(1)
              if N <= 1:
                  events.append(Event("start", ts, stack[i]))
                  new_confirmed.append(True)
              else:
                  new_confirmed.append(False)
 
          prev_stack = stack
          streak_counts = new_streaks
          confirmed = new_confirmed
      return events
 
ADDITIONAL DISCUSSION TOPIC (verbal)
--------------------------------------
Prefix vs Postfix (Suffix) Comparison:
  - Prefix (current approach): Compare from root downward. Coarse-grained.
  - Postfix (leaf-first): More precise for profiling since leaf functions change most.
  - Example: Stack A: main→handler→parse→foo vs Stack B: main→handler→parse→bar
    Prefix: 3-layer match, may consider same
    Postfix: Leaf differs clearly

CODING #4 - DISTRIBUTED MODE AND MEDIAN
Research Engineer, Software Engineer
================================================================================
 
BASIC PROBLEM
-------------
Given:
- A very large dataset distributed across multiple machines (typically 10 workers)
- Pre-built interface functions: send(worker_id, data) and recv()
- Each machine has a portion of the dataset stored locally
 
Find the MODE (most frequent value) of the entire dataset.
 
Critical Constraint: Avoid having all machines simultaneously send data to a
single machine (creates severe reception delays).
 
Interface:
  def send(worker_id, data): pass
  def recv(): pass
 
Dataset: Contains mostly unique integers, one integer appears twice on different
machines.
 
DETAILED IMPLEMENTATION: MODE
------------------------------
Phase 1: Local Processing
  from collections import Counter
  def phase1_local_count(local_data):
      return Counter(local_data)
 
Phase 2: Distributed Shuffle (Key Innovation)
  - Each worker redistributes local counts using: target_worker = key % num_workers
  - Ensures all identical keys end up on the same worker
  - All workers send and receive simultaneously (no bottleneck)
 
  def phase2_shuffle_counts(local_counter, worker_id, num_workers):
      buckets = [[] for _ in range(num_workers)]
      for key, count in local_counter.items():
          target_worker = key % num_workers
          buckets[target_worker].append((key, count))
      for target_id in range(num_workers):
          if target_id != worker_id:
              send(target_id, buckets[target_id])
      received_data = []
      for _ in range(num_workers - 1):
          data = recv()
          received_data.extend(data)
      received_data.extend(buckets[worker_id])
      return received_data
 
Phase 3: Local Aggregation
  - Each worker aggregates all counts for its assigned keys
  - Gets local top-1 (sufficient because keys are disjoint)
 
Phase 4: Global Reduction
  - All workers send their local top-1 to worker 0
  - Worker 0 computes global mode
 
NOTE on top-1 sufficiency: After Phase 2, each worker owns a DISJOINT set of keys.
The global mode must be the maximum among all workers' local maximums.
 
FOLLOW-UP: FINDING THE MEDIAN
------------------------------
Key Insight: Use distributed quickselect algorithm.
 
Phase 1&2: Build distributed histogram (reuse mode phases)
 
Phase 3: Distributed Quickselect (binary search)
  - Worker 0 coordinates: picks pivot = (low + high) // 2
  - Each worker counts values: < pivot, == pivot, > pivot
  - Aggregate on worker 0, decide next search range
  - Repeat until low == high == median
 
  def distributed_quickselect(local_histogram, worker_id, num_workers, target_position):
      # Find global min/max
      # Binary search:
      while low < high:
          if worker_id == 0:
              pivot = (low + high) // 2
              for i in range(1, num_workers): send(i, pivot)
          else:
              pivot = recv()
 
          count_less = sum(count for val, count in local_histogram.items() if val < pivot)
          count_equal = local_histogram.get(pivot, 0)
          count_greater = sum(count for val, count in local_histogram.items() if val > pivot)
 
          # Worker 0 aggregates and broadcasts new range
          # ...
      return low
 
ADDITIONAL DISCUSSION TOPICS (verbal)
--------------------------------------
1. Handling Data Skew
   - Use MurmurHash/CityHash instead of key % num_workers for better distribution
   - Dynamic load balancing
 
2. Fault Tolerance
   - Checkpointing after each phase
   - Heartbeats, replication, retry mechanism
 
3. Optimizing for Different Data Sizes
   - Small: Gather all to one machine
   - Very large: Multi-level aggregation, approximate algorithms (Count-Min Sketch)
   - Streaming: Two-heap approach for median
 
4. Communication Optimization
   - Async communication, batching, tree-based reduction
   - Deadlock prevention: Interleave send/recv operations
 
5. Extending to Other Statistics
   - Top K: Send top K from each worker after shuffle
   - Percentiles: Similar to median, distributed quantile sketch (T-Digest)
   - Variance: Parallel variance formula, Welford's method
   - Distinct count: HyperLogLog

  def process_single_image(args):
      image_path, transform_path, output_path = args
      with open(transform_path) as f:
          data = json.load(f)
      transformations = data.get('transformations', [])
      image = Image.open(image_path)
      result = image.copy()
      for transform in transformations:
          result = apply_transformation(result, transform)
      os.makedirs(os.path.dirname(output_path), exist_ok=True)
      result.save(output_path)
      image.close(); result.close()
      return output_path
 
  def process_images_parallel(image_dir, transformation_dir, output_dir, get_output_path):
      image_files = [f for f in Path(image_dir).iterdir()
                     if f.suffix.lower() in ('.png', '.jpg', '.jpeg')]
      transform_files = list(Path(transformation_dir).glob('*.json'))
      work_items = []
      for transform_file in transform_files:
          for image_file in image_files:
              output_path = get_output_path(str(image_file), str(transform_file))
              work_items.append((str(image_file), str(transform_file), output_path))
      with ProcessPoolExecutor() as executor:
          results = list(executor.map(process_single_image, work_items))
 
FOLLOW-UP: Threading vs Multiprocessing
-----------------------------------------
ProcessPoolExecutor:
  - Best for CPU-bound tasks
  - Bypasses GIL (separate interpreters)
  - Higher overhead (process creation)
 
ThreadPoolExecutor:
  - Best for I/O-bound tasks
  - Limited by GIL for Python code
  - Lower overhead
 
Image processing (blur, rotate, scale) is CPU-intensive → ProcessPoolExecutor wins.
GIL is released during I/O, but computation bottleneck makes multiprocessing better.

SYSTEM DESIGN #11 - DISTRIBUTED MODEL DEPLOYMENT
Software Engineer
================================================================================
 
PROBLEM STATEMENT
-----------------
Design a system that efficiently downloads and distributes a large ML model
(e.g., 500GB) from external storage to all GPU workers in a data center cluster.
 
Given Constraints:
  - Model size: 500GB
  - Workers: 100-1000 GPU workers
  - External bandwidth: 10 Gbps (download from S3/GCS)
  - Internal bandwidth per worker: 10 Gbps full-duplex
  - Objective: Minimize total time to deploy to ALL workers
  - Reliability: Handle worker failures
 
CAPACITY ESTIMATION
--------------------
  - Transfer 500GB at 10 Gbps: 500×8 / 10 = 400 seconds
  - Chunk size (1 GB): 1×8 / 10 = 0.8 seconds
  - Number of chunks: 500
 
DISTRIBUTION STRATEGY COMPARISON
---------------------------------
Approach 1: Sequential (Naive — everyone downloads from external)
  - Each worker: 10 Gbps / 100 = 0.1 Gbps per worker
  - Time: 500×8 / 0.1 = 40,000s ≈ 11 HOURS. Unacceptable.
 
Approach 2: Binary Tree Distribution
  - Each parent splits bandwidth across 2 children → 5 Gbps per child
  - Time: 500×8 / 5 + 7 levels × 1.6s = 811s ≈ 13.5 min
  - For 1000 workers: ≈ 13.6 min (logarithmic in N)
 
Approach 3: Pipeline Distribution (BEST for ~100 workers)
  W1 → W2 → W3 → ... → W100
  Phase 1: Download all 500 chunks through W1 = 400s
  Phase 2: Last chunk propagates: 99 hops × 0.8s = 79.2s
  Total: 479s ≈ 8 MINUTES ✓
 
Comparison:
  Strategy         | 100 Workers | 1000 Workers | Complexity
  Sequential       | 11 hours    | 111 hours    | O(N×S)
  Binary Tree      | 13.5 min    | 13.6 min     | O(S + C×logN)
  Pipeline         | 8 min       | 20 min       | O(S + N×C)
 
Recommendation: Pipeline for ~100 workers; binary tree for large N.
 
PHASE 3: API DESIGN
--------------------
Coordinator:
  POST /deployments → {deployment_id, status, estimated_time_seconds}
  GET /deployments/{id} → {status, progress, workers_complete, workers_failed}
 
Worker-to-Worker (gRPC):
  service ChunkTransfer {
    rpc StreamChunks(stream ChunkRequest) returns (stream ChunkData);
  }
 
PHASE 4: REQUEST FLOW
----------------------
Phase 1: Initialization
  - Coordinator fetches manifest (pre-computed checksums)
  - Assigns pipeline topology: [W1, W2, ..., W100]
 
Phase 2: Seeding
  - W1 downloads chunk, immediately streams to W2 (full-duplex)
  - W1: Download chunk N, send chunk N-1 to W2 simultaneously
 
Phase 3: Steady-State
  # Each worker runs two threads
  Thread 1 (receive): Download chunks, verify SHA256, write to disk, report progress
  Thread 2 (send): Wait until chunk available, send to downstream
 
Phase 4: Completion
  - Verify global checksum, report SUCCESS/FAILURE to coordinator
 
FAILURE HANDLING
-----------------
Worker Failure Mid-Pipeline:
  Before: W1 → W2 → W3 → W4 → W5
  After:  W1 → W2 ────→ W4 → W5
 
  Recovery:
  1. Coordinator detects failure (missed heartbeats over 5-10s)
  2. Updates topology: W2's downstream = W4
  3. W4 requests missing chunks from W2
 
Coordinator Failure:
  - 3-5 replicas with etcd leader election
 
Corrupted Chunks:
  - SHA256 checksum per chunk
  - Retry with exponential backoff (max 3 attempts)
 
RACK-AWARE TOPOLOGY
--------------------
Order pipeline by rack to minimize cross-rack hops:
  Naive:      W1(R1)→W4(R2)→W2(R1)→... (5 cross-rack hops)
  Rack-aware: W1(R1)→W2(R1)→W3(R1)→W4(R2)→... (1 cross-rack hop)
 
SCALING
--------
  100 workers:   479s (8 min)   — 400s download + 79s propagation
  1,000 workers: 1,199s (20 min) — propagation starts to dominate
  10,000+:       Consider tree-of-pipelines hybrid

Implement a simplified in-memory database. 4 progressive levels.
 
INPUT/OUTPUT FORMAT
--------------------
Input: List of queries (array of string arrays)
Output: Array of strings (returned values of all queries)
Note: Operations given in strictly increasing timestamp order.
 
LEVEL 1: BASIC OPERATIONS
--------------------------
Operations:
 
SET <timestamp> <key> <field> <value>
  - Insert/update field-value in record; returns ""
 
COMPARE_AND_SET <timestamp> <key> <field> <expectedValue> <newValue>
  - Update field to newValue only if current == expectedValue
  - Returns "true" if updated, "false" otherwise
 
COMPARE_AND_DELETE <timestamp> <key> <field> <expectedValue>
  - Remove field only if current value == expectedValue
  - Returns "true" if removed, "false" otherwise
 
GET <timestamp> <key> <field>
  - Returns value or "" if not found
 
Example:
  SET 0 A B 4 → ""
  SET 1 A C 6 → ""
  COMPARE_AND_SET 2 A B 4 9 → "true"  (B was 4, now 9)
  COMPARE_AND_SET 3 A C 4 9 → "false" (C is 6, not 4)
  COMPARE_AND_DELETE 4 A C 6 → "true"
  GET 5 A C → ""
  GET 6 A B → "9"
 
LEVEL 2: FILTER-BASED DISPLAY
-------------------------------
SCAN <timestamp> <key>
  - Returns all fields sorted lexicographically
  - Format: "field1(value1), field2(value2), ..."
  - Returns "" if record doesn't exist
 
SCAN_BY_PREFIX <timestamp> <key> <prefix>
  - Returns fields starting with prefix, sorted lexicographically
 
Example:
  SET 1 A BC 1 → ""
  SET 2 A BD 2 → ""
  SET 3 A C 3 → ""
  SCAN_BY_PREFIX 4 A B → "BC(1), BD(2)"
  SCAN 5 A → "BC(1), BD(2), C(3)"
 
LEVEL 3: TTL (TIME-TO-LIVE)
----------------------------
SET_WITH_TTL <timestamp> <key> <field> <value> <ttl>
  - Field exists during [timestamp, timestamp + ttl)
  - Returns ""
 
All previous operations now respect TTL expiration.
 
Example:
  SET_WITH_TTL 1 A BC 1 9 → ""  (expires at 10)
  SET_WITH_TTL 5 A BC 2 10 → "" (expires at 15, overwrites)
  SET 6 A BD 3 → ""  (no TTL)
  SCAN_BY_PREFIX 14 A B → "BC(2), BD(3)"
  SCAN_BY_PREFIX 15 A B → "BD(3)"   (BC expired)
 
LEVEL 4: BACKUP AND RESTORE
----------------------------
BACKUP <timestamp> <backup_at_timestamp>
  - Saves snapshot of all records
  - Returns count of valid records (with at least one non-expired field)
  - backup_at_timestamp is the snapshot identifier
 
RESTORE <timestamp> <restore_at_timestamp>
  - Restores from backup
  - If no exact backup at restore_at_timestamp, use latest backup <= that timestamp
  - TTL recalculated: new_expiry = restore_timestamp + remaining_TTL_at_backup
  - Returns ""
 
Example:
  SET_WITH_TTL 1 A x 10 10 → ""  (A.x expires at 11)
  SET 2 B y 20 → ""
  BACKUP 5 5 → "2"
  SET 6 A x 99 → ""  (overwrite)
  BACKUP 8 8 → "2"
  COMPARE_AND_DELETE 9 B y 20 → "true"
  RESTORE 12 7 → ""  (no backup at 7, use backup at 5)
  GET 13 A x → "10"  (restored value; remaining TTL was 6, new expiry = 12+6=18)
  GET 14 B y → "20"  (restored from backup)
  SCAN 19 A → ""     (A.x expired at 18)

from dataclasses import dataclass
from typing import Dict, List, Optional, Tuple
 
@dataclass
class FieldState:
    value: int
    expires_at: Optional[int] = None
 
RecordState = Dict[str, FieldState]
 
class InMemoryDBLevel1:
    def __init__(self):
        self.records: Dict[str, RecordState] = {}
 
    def is_alive(self, field: FieldState, timestamp: int) -> bool:
        return field.expires_at is None or timestamp < field.expires_at
 
    def get_or_create_record(self, key):
        if key not in self.records:
            self.records[key] = {}
        return self.records[key]
 
    def read_live_field(self, timestamp, key, field):
        record = self.records.get(key)
        if record is None: return None
        state = record.get(field)
        if state is None: return None
        if self.is_alive(state, timestamp): return state
        del record[field]
        if not record: del self.records[key]
        return None
 
    def set(self, timestamp, key, field, value):
        record = self.get_or_create_record(key)
        record[field] = FieldState(value=value, expires_at=None)
        return ''
 
    def compare_and_set(self, timestamp, key, field, expected_value, new_value):
        current = self.read_live_field(timestamp, key, field)
        if current is None or current.value != expected_value: return 'false'
        self.records[key][field] = FieldState(value=new_value, expires_at=current.expires_at)
        return 'true'
 
    def compare_and_delete(self, timestamp, key, field, expected_value):
        current = self.read_live_field(timestamp, key, field)
        if current is None or current.value != expected_value: return 'false'
        record = self.records[key]
        del record[field]
        if not record: del self.records[key]
        return 'true'
 
    def get(self, timestamp, key, field):
        current = self.read_live_field(timestamp, key, field)
        return str(current.value) if current else ''
 
class InMemoryDBLevel2(InMemoryDBLevel1):
    def list_live_fields(self, timestamp, key):
        record = self.records.get(key)
        if record is None: return []
        to_delete = []
        entries = []
        for field, state in record.items():
            if self.is_alive(state, timestamp): entries.append((field, state))
            else: to_delete.append(field)
        for field in to_delete: del record[field]
        if not record: del self.records[key]
        entries.sort(key=lambda pair: pair[0])
        return entries
 
    def scan(self, timestamp, key):
        return ', '.join(f'{field}({state.value})'
                        for field, state in self.list_live_fields(timestamp, key))
 
    def scan_by_prefix(self, timestamp, key, prefix):
        return ', '.join(f'{field}({state.value})'
                        for field, state in self.list_live_fields(timestamp, key)
                        if field.startswith(prefix))
 
class InMemoryDBLevel3(InMemoryDBLevel2):
    def set_with_ttl(self, timestamp, key, field, value, ttl):
        record = self.get_or_create_record(key)
        record[field] = FieldState(value=value, expires_at=timestamp + ttl)
        return ''
 
@dataclass
class SnapshotField:
    value: int
    remaining_ttl: Optional[int]
 
@dataclass
class SnapshotRecord:
    fields: Dict[str, 'SnapshotField']
 
class InMemoryDBLevel4(InMemoryDBLevel3):
    def __init__(self):
        super().__init__()
        self.backups: Dict[int, Dict] = {}
 
    def backup(self, timestamp, backup_at_timestamp):
        snapshot = {}
        for key, record in self.records.items():
            snapshot_fields = {}
            for field, state in record.items():
                if not self.is_alive(state, timestamp): continue
                remaining_ttl = None if state.expires_at is None else state.expires_at - timestamp
                snapshot_fields[field] = SnapshotField(value=state.value, remaining_ttl=remaining_ttl)
            if snapshot_fields:
                snapshot[key] = SnapshotRecord(fields=snapshot_fields)
        self.backups[backup_at_timestamp] = snapshot  # deep copy needed in practice
        return str(len(snapshot))
 
    def restore(self, timestamp, restore_at_timestamp):
        candidates = [ts for ts in self.backups.keys() if ts <= restore_at_timestamp]
        if not candidates: return ''
        source_ts = max(candidates)
        source_snapshot = self.backups[source_ts]
        self.records = {}
        for key, snapshot_record in source_snapshot.items():
            record = {}
            for field, sf in snapshot_record.fields.items():
                expires_at = None if sf.remaining_ttl is None else timestamp + sf.remaining_ttl
                record[field] = FieldState(value=sf.value, expires_at=expires_at)
            if record:
                self.records[key] = record
        return ''

class RecipeManager:
    def __init__(self):
        self.recipes = {}       # recipe_id -> {id, history: [{name, ingredients}]}
        self.users = {}         # user_id -> user data
        self.name_to_id = {}    # lowercase name -> recipe_id
 
    def add_recipe(self, recipe_id, name, ingredients):
        if recipe_id in self.recipes: return False
        if name.lower() in self.name_to_id: return False
        self.recipes[recipe_id] = {
            'id': recipe_id,
            'history': [{'name': name, 'ingredients': ingredients.copy()}]
        }
        self.name_to_id[name.lower()] = recipe_id
        return True
 
    def get_recipe(self, recipe_id):
        if recipe_id not in self.recipes: return None
        recipe = self.recipes[recipe_id]
        current = recipe['history'][-1]
        return {'id': recipe_id, 'name': current['name'], 'ingredients': current['ingredients'].copy()}
 
    def _sort_recipes(self, recipes):
        def sort_key(r):
            count = len(r['ingredients'])
            rid = r['id']
            if rid.isdigit(): return (count, 0, int(rid), "")
            else: return (count, 1, 0, rid)
        return sorted(recipes, key=sort_key)
 
    def search_recipes(self, query):
        q = query.lower()
        results = []
        for rid, recipe in self.recipes.items():
            current = recipe['history'][-1]
            if q in current['name'].lower():
                results.append({'id': rid, 'name': current['name'],
                               'ingredients': current['ingredients'].copy()})
        return self._sort_recipes(results)
 
    def add_user(self, user_id, username):
        if user_id in self.users: return False
        self.users[user_id] = {'id': user_id, 'username': username}
        return True
 
    def edit_recipe(self, user_id, recipe_id, name, ingredients):
        if user_id not in self.users: return False
        if recipe_id not in self.recipes: return False
        existing_id = self.name_to_id.get(name.lower())
        if existing_id is not None and existing_id != recipe_id: return False
        old_name = self.recipes[recipe_id]['history'][-1]['name']
        del self.name_to_id[old_name.lower()]
        self.recipes[recipe_id]['history'].append({'name': name, 'ingredients': ingredients.copy()})
        self.name_to_id[name.lower()] = recipe_id
        return True
 
    def get_recipe_history(self, recipe_id):
        if recipe_id not in self.recipes: return None
        return [{'version': i+1, 'name': v['name'], 'ingredients': v['ingredients'].copy()}
                for i, v in enumerate(self.recipes[recipe_id]['history'])]
 
    def rollback_recipe(self, user_id, recipe_id, version):
        if user_id not in self.users: return False
        if recipe_id not in self.recipes: return False
        history = self.recipes[recipe_id]['history']
        if version < 1 or version > len(history): return False
        old_version = history[version - 1]
        current_name = history[-1]['name']
        if old_version['name'].lower() != current_name.lower():
            existing_id = self.name_to_id.get(old_version['name'].lower())
            if existing_id is not None and existing_id != recipe_id: return False
        del self.name_to_id[current_name.lower()]
        history.append({'name': old_version['name'], 'ingredients': old_version['ingredients'].copy()})
        self.name_to_id[old_version['name'].lower()] = recipe_id
        return True

from typing import Optional

class Task:
    def __init__(self, task_id, name, priority):

        self.name = name
        self.priority = priority
 
class User:
    def __init__(self, user_id, quota):

        self.quota = quota
        self.assignments: list[Assignment] = []
 
class TaskManagementSystem:
    def __init__(self):
        self.tasks: dict[str, Task] = {}
        self.task_counter = 0
        self.creation_order: dict[str, int] = {}
        self.users: dict[str, User] = {}
 
    # Level 1
    def add_task(self, timestamp, name, priority):
        self.task_counter += 1
        task_id = f"task_id_{self.task_counter}"
        self.tasks[task_id] = Task(task_id, name, priority)
        self.creation_order[task_id] = self.task_counter
        return task_id
 
    def update_task(self, timestamp, task_id, name, priority):
        if task_id not in self.tasks: return False
        self.tasks[task_id].name = name
        self.tasks[task_id].priority = priority
        return True
 
    def get_task(self, timestamp, task_id):
        if task_id not in self.tasks: return None
        t = self.tasks[task_id]
        return f'{{"name":"{t.name}","priority":{t.priority}}}'
 
    # Level 2
    def search_tasks(self, timestamp, name_filter, max_results):
        if max_results <= 0: return []
        matching = [tid for tid, t in self.tasks.items() if name_filter in t.name]
        matching.sort(key=lambda tid: (-self.tasks[tid].priority, self.creation_order[tid]))
        return matching[:max_results]
 
    def list_tasks_sorted(self, timestamp, limit):
        if limit <= 0: return []
        task_ids = list(self.tasks.keys())
        task_ids.sort(key=lambda tid: (-self.tasks[tid].priority, self.creation_order[tid]))
        return task_ids[:limit]
 
    # Level 3
    def add_user(self, timestamp, user_id, quota):
        if user_id in self.users: return False
        self.users[user_id] = User(user_id, quota)
        return True
 
    def assign_task(self, timestamp, task_id, user_id, finish_time):
        if task_id not in self.tasks or user_id not in self.users: return False
        user = self.users[user_id]
        active = sum(1 for a in user.assignments
                    if a.start_time <= timestamp < a.finish_time and not a.completed)
        if active >= user.quota: return False
        user.assignments.append(Assignment(task_id, user_id, timestamp, finish_time))
        return True
 
    def get_user_tasks(self, timestamp, user_id):
        if user_id not in self.users: return []
        user = self.users[user_id]
        active = [a for a in user.assignments
                  if a.start_time <= timestamp < a.finish_time and not a.completed]
        active.sort(key=lambda a: (a.finish_time, a.start_time))
        return [a.task_id for a in active]
 
    # Level 4
    def complete_task(self, timestamp, task_id, user_id):
        if task_id not in self.tasks or user_id not in self.users: return False
        user = self.users[user_id]
        active = [a for a in user.assignments
                  if a.task_id == task_id
                  and a.start_time <= timestamp < a.finish_time
                  and not a.completed]
        if not active: return False
        active.sort(key=lambda a: a.start_time)
        active[0].completed = True
        return True
 
    def get_overdue_assignments(self, timestamp, user_id):
        if user_id not in self.users: return []
        user = self.users[user_id]
        overdue = [a for a in user.assignments
                   if a.finish_time <= timestamp and not a.completed]
        overdue.sort(key=lambda a: (a.finish_time, a.start_time))
        return [a.task_id for a in overdue]
